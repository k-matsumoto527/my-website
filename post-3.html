REST APIとは何か？初心者向け解説

システム間のデータのやり取りに使われるREST API。HTTPメソッドの意味から、実際のリクエスト・レスポンスの形まで解説します。

REST APIの概要

REST（Representational State Transfer）は、Web APIを設計するためのアーキテクチャスタイルです。URLでリソースを表し、HTTPメソッドで操作の種類を表現するのが基本です。

HTTPメソッドの使い分け

メソッド 操作 例
GET データを取得 記事一覧を取得
POST データを作成 新規記事を投稿
PUT データを更新 記事を上書き更新
DELETE データを削除 記事を削除

リクエストとレスポンスの例

GET /posts/1 というリクエストを送ると、サーバーはJSON形式でデータを返します。

{
  "id": 1,
  "title": "はじめてのDockerコンテナ入門",
  "date": "2026-05-20",
  "tag": "インフラ"
}

新規作成は POST /posts にリクエストボディを付けて送ります。レスポンスには作成されたリソースが返ってきます。REST APIをマスターすると、フロントエンドとバックエンドの連携がぐっとスムーズになります。
