GitのブランチとPull Requestの使い方

チーム開発で必須のGitブランチ運用。main ブランチを守りながら機能を追加するための基本的なフローをまとめました。

ブランチを切る理由

ブランチを使うと、本番コード（main）に影響を与えずに新機能の開発やバグ修正を進められます。作業が終わったらPull Request（PR）を出してレビューを受け、問題なければmainにマージします。

基本的なフロー

# 最新のmainを取得
git checkout main
git pull origin main

# 作業用ブランチを作成・切り替え
git checkout -b feature/add-login

# 変更をコミット
git add .
git commit -m "ログイン機能を追加"

# リモートにプッシュ
git push origin feature/add-login

Pull Requestを出す

プッシュ後にGitHubを開くと「Compare & pull request」ボタンが表示されます。変更内容と背景を書いてPRを作成し、チームメンバーにレビューを依頼しましょう。

マージ後のブランチ削除

マージが完了したらブランチは不要になります。GitHub上の「Delete branch」ボタンで削除し、ローカルでも以下のコマンドでクリーンアップしましょう。

git checkout main
git pull origin main
git branch -d feature/add-login
