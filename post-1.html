はじめてのDockerコンテナ入門

Dockerを使うと、アプリの実行環境をコンテナとして丸ごとパッケージできます。「自分のPCでは動くのに本番では動かない」という問題を根本から解決してくれる技術です。

コンテナとは？

コンテナは、アプリとその依存関係（ライブラリ・ランタイムなど）をひとまとめにした軽量な実行単位です。仮想マシン（VM）と違い、OSカーネルをホストと共有するため起動が高速で、リソース消費も少なくなります。

基本的なコマンド

まずはイメージを取得して、コンテナを起動してみましょう。

# イメージを取得
docker pull nginx

# コンテナを起動（ポート80を公開）
docker run -d -p 8080:80 nginx

# 起動中のコンテナを確認
docker ps

Dockerfileを書く

自分のアプリをコンテナ化するには Dockerfile を用意します。

FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
CMD ["node", "index.js"]

あとは docker build -t my-app . でイメージをビルドするだけです。チームで同じイメージを共有することで、環境差異をなくすことができます。
